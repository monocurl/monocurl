- bytecode: contains the enum of the Monocurl VM bytecode
  - src/lib.rs: Instruction / prototype definitions plus SectionBytecode metadata (flags, instruction annotations, source file name, imported-bundle display index)
- cli: binary crate that contains the gluing logic for the monocurl cli
- compiler: converts AST into bytecode, performs static analysis
  - src/compiler/mod.rs: compiler module root; shared symbol/compile-result types, the `compile(...)` entrypoint, compiler-frame state, and bundle/section emission helpers
  - src/compiler/cursor.rs: cursor-visible symbol enumeration for autocomplete plus root-reference tracking metadata used by signature hints
  - src/compiler/statements.rs: statement and control-flow lowering (`let`/`var`/`mesh`/`param`, loops, `if`, `return`, `break`, `continue`, `play`)
  - src/compiler/expressions.rs: expression lowering, identifier/property/subscript access, literals, unary/binary ops, and lambda/operator/native invocation bytecode emission
  - src/compiler/closures.rs: lambda/operator/block/anim lowering, closure-frame setup/teardown, capture registration, and implicit block `_` accumulation. `block { ... }` bodies still default to `[]`, while ordinary block-bodied lambdas now fall through to an explicit runtime error anchored to the full lambda span
  - src/compiler/free_vars.rs: free-variable and free-lvalue walkers used for lambda/block capture analysis
  - src/compiler/stateful.rs: compiler-side stateful-expression lowering and validation helpers
  - src/compiler/tests.rs: compiler unit tests split out of the module root
  - src/compiler/warnings.rs: AST walker for compile-time warnings about useless expression statements with no syntactic side effects
- executor: takes bytecode and scene state and executes it
  - src/error.rs: ExecutorError enum plus RuntimeError / RuntimeCallFrame (raw execution error, selected span, bounded recovered callstack)
  - src/heap.rs: thread-local virtual heap for heap-backed values/lvalues; owns `VRc`/`VWeak`, slot refcounts/free-list reuse, raw snapshot wrappers for cacheing, and refcount inhibition used while cloning/restoring cached executor state
  - src/executor/mod.rs: Executor struct and public executor API entrypoints, plus seek/result enums and NativeFunc/NativeFuture types
  - src/executor/dispatch.rs: main instruction dispatch loop for bytecode op execution, including stack reads/writes, invocation dispatch, control flow, and dereference helper logic
  - src/executor/memory.rs: periodic executor memory guard for instruction stepping; now checks both process RSS and virtual-heap slot count against hard limits on a throttled cadence
  - src/executor/runtime_error.rs: recovered runtime error span/callstack construction from stack ancestry and saved call frames, including play-aware recovery and outermost-first displayed callstacks so spawning `play` sites appear before inner imported helper frames; plus public helpers to record an `ExecutorError` either at the current execution context or explicitly at the root stack when a subsystem such as UI scene snapshotting needs the diagnostic to anchor near the current top-level playback position. Those record helpers now return the `RuntimeError` they stored, so callers can react directly without re-inspecting executor state. Span recovery now skips empty sentinel/import annotations, falls back to the most recent real root-module span across earlier sections when the current section is empty, and only then uses section-wide annotation extents, so post-execution/root-recorded errors do not collapse to `0..1` just because execution is parked at an empty slide tail
  - src/executor/cacheing.rs: per-section execution cache used for rebasing seeks plus user-visible slide duration metadata, including exact cached slide durations and invalidation-aware minimum duration lower bounds
  - src/executor/ops.rs: binary/unary operations with int→float→complex type promotion plus recursive list linear ops (scalar multiply, elementwise add, negate); owns BinOp enum (including stateful `and`/`or` lifting). Division now consistently raises runtime `division by zero` across integer, float, and complex arithmetic instead of producing infinities/NaNs. Intercepts Value::Stateful operands to build StatefulNode::BinaryOp/UnaryNeg/Not lifted nodes instead of evaluating eagerly
  - src/executor/invoke.rs: async lambda/operator/native invocation, call frame setup, labeled invocations, isolated eager lambda execution with trace-parent stack links, and operator invocation preservation. Operator calls now always materialize `InvokedOperator` metadata (even with unlabeled args) so operand attributes remain reachable; live/stateful cache writes now recursively materialize nested wrappers before storing them, so later scene capture and mesh reads see concrete values instead of stale `live function` / `live operator` placeholders; `exec_convert_to_live_operator` is retained as a compatibility no-op for that stack shape
  - src/executor/lerp.rs: general lerp(a, b, t) for Monocurl values — handles numbers, same-lambda InvokedFunction arg interpolation, InvokedOperator unmodified embeds, same-topology mesh interpolation for animation followers, and list-length mismatch diagnostics phrased in list terms rather than `n`-vector jargon
  - src/executor/anim.rs: section advancement, seek_primitive_anim (async, yields between instructions), playback stepping/seek_to, play/spawn/bake, primitive target flattening/deduction with ancestor-based implicit leader selection, leader lock handling, and double-play guard via AnimBlock.already_played. `play` now elides live function/operator wrappers before dispatch, so animation operators such as `slow{...}` / `fast{...}` / `delay{...}` can feed primitive anims or anim blocks directly. Primitive `Lerp` bake now resolves the final target set, snapshots each original destination, optionally runs an embed callable `(start, dst) -> [a, b, state]`, stores the baked `[a, b, state]` triple, and at step time either uses the provided custom lerp callable `(a, b, state, t)` or the generic executor lerp; completion always snaps followers to the original baked destination rather than reusing the custom lerp result. Internal animation callback invocation now materializes stateful/live arguments before eager lambda/operator calls, so follower helpers such as `Fade` / `Grow` / custom animation lerps can inspect reactive mesh destinations without tripping the direct-stateful-copy guard
  - src/executor/access.rs: subscript/attribute (mutable + immutable), assign, append, heap-backed COW for shared list/map containers and element slots, WeakLvalue handling, leader passthrough for nested subscript/attribute mutation; assignment expressions now yield the assigned rhs value (so chained assignments propagate values rather than lvalues). Mesh-leader reassignment now seeds an empty follower from the prior leader value before overwriting, so first-play animations like `Trans` can still start from the pre-assignment mesh. Value::Stateful attribute access destructures LabeledCall/LabeledOperatorCall nodes; stateful assignment to non-mesh-leader is a runtime error
  - src/state.rs: ExecutionState (execution stacks, ghost stack lineage metadata, leaders, explicit active params, primitive anims, ephemeral_pool, structured runtime errors, last runtime-error stack context), ExecutionStack (var stack, IP, call stack, labels, control parent + trace parent), BakedPrimitiveAnim (target leaders + baked original destinations + embedded start/end/state triples for primitive interpolation), LeaderEntry/ActiveParam metadata (declared name plus leader/follower refs). sync_all_leaders now clones leader values into follower context by converting embedded Stateful values to follower-read mode and bumping follower versions. Monotonic stack IDs and primitive animation IDs; finished stacks remain as ghosts so ancestry checks still work after free.
  - src/scene_snapshot.rs: shared background/camera/visible-mesh capture for any frontend. Validates scene leaders by eliding live/stateful wrappers, parses camera/background values into concrete snapshot structs with list-length-aware diagnostics, now treats the public camera shape as `position` + `forward` + `up` + clip range (while still accepting legacy `look_at` data for compatibility), collects visible mesh trees, and returns a `Result` from stable snapshot capture so callers can see directly whether scene capture recorded a root-position runtime error
  - src/value/mod.rs: Value enum plus heap-backed `Lvalue(VRc)` / `WeakLvalue(VWeak)` handles, `InstructionPointer`, and value submodule wiring
  - src/value/helpers.rs: Value helper methods for truthiness, lvalue/wrapper elision (including stateful reads), lvalue access, and runtime type naming
  - src/value/equality.rs: structural equality for values, primitive anim candidates, and reactive stateful trees
  - src/value/container.rs: List (SmallVec of heap keys) and Map (hashable keys to heap keys plus insertion_order for deterministic iteration) with retain/release-aware clone/drop behavior
  - src/value/lambda.rs: Lambda (captures, defaults, IP) and Operator wrapper
  - src/value/anim_block.rs: AnimBlock (captures + IP for coroutine-style animation blocks, already_played: Rc<Cell<bool>> to prevent double-play)
  - src/value/leader.rs: Leader struct (kind: LeaderKind, leader/follower heap keys, detached status via last_modified_stack, active animation lock, leader_version + follower_version u64 counters for stateful cache invalidation) for mesh/param variables
  - src/value/invoked_function.rs: InvokedFunction cache/recompute wrapper for labeled function calls; cache hits and misses now both store fully materialized values so nested live wrappers do not leak back out through later reads
  - src/value/invoked_operator.rs: InvokedOperator (operator + operand + labels + unmodified identity embed + cached modified value); extract_operator_result and build_invoked_operator helpers. Both cached branches now store fully materialized initial/modified values for the same reason as InvokedFunction
  - src/value/primitive_anim.rs: PrimitiveAnim enum (Lerp, Set, Wait) with explicit candidate-tree support for targeted sync animations; `Lerp` carries candidate hints, duration, optional rate callable, optional embed callable, and optional custom lerp callable so primitive animations can bake mapped interpolation state before playback starts
  - src/value/stateful.rs: Stateful (roots: Vec<HeapKey> + root: StatefulNode + read_kind leader/follower mode + version-based cache) and StatefulNode enum for reactive dependency graphs; variants: LeaderRef, Constant, LabeledCall, LabeledOperatorCall, Subscript; constructor dedups root heap cells by key; labeled-call/operator args are stored as heap keys to support mutable attribute access; async Executor::eval_stateful evaluates the full node tree against either leader or follower versions depending on context
  - src/time.rs: Timestamp (slide + time offset)
- exporter: coordinates process of exporting a scene into a video
- geo: helper routines for execution (a lot of lib monocurl will reference these routines).
  - src/mesh.rs: core mesh structs (`Dot`, `Lin`, `Tri`, `Mesh`, `Uniforms`), lightweight per-mesh render metadata such as alpha/texture/z-order, and topology invariant validation helpers; topology is expected to be authored by constructors/ops rather than inferred from geometry
- libtess2: safe Rust wrapper over a vendored copy of the upstream libtess2 tessellator
  - src/lib.rs: `Float3`-based contour input, tessellation options/error types, opt-in contour normalization for degenerate/redundant loop sets before triangulation, and safe triangle output extraction from `TESStesselator`
  - build.rs: compiles the vendored upstream C sources from `upstream/Source` and links them into the Rust crate
  - upstream/: git-cloned `https://github.com/memononen/libtess2` source tree with `.git` removed and `LICENSE.txt` retained
- lexer: lexing utilites, contains token definition
  - src/lexer.rs: actually lexes a stream of characters
  - src/token.rs: contains an enum of all of the tokens and helper functions related to them
- parser: transforms lexed source file into an AST
  - src/ast.rs: contains structs related to all of the types of ast nodes
  - src/parser/mod.rs: parser module root; shared parser state/diagnostic structs, bracket precomputation, token-reading helpers, and top-level `Parser` / `SectionParser` definitions
  - src/parser/predicate.rs: token/state predicates used by the tolerant `try_all!` parser machinery and cursor-possibility emission
  - src/parser/statements.rs: section/statement-list parsing plus statement bodies, declarations, loop/if/play/return parsing, and identifier declarations
  - src/parser/expressions.rs: precedence parser, postfix/property/subscript/invocation parsing, and lambda/block/anim expression parsing
  - src/parser/literals.rs: string/numeric/vector/map literal parsing helpers
  - src/parser/imports.rs: import DFS/preparse logic, file splitting into sections, cache-aware bundle assembly, and the public parse entrypoint
  - src/parser/tests.rs: parser unit tests split out of the module root
- renderer: given a state snapshot, actually renders it via platform specific shaders
- stdlib: actual lib monocurl routine implementations. depends on executor for Value types
  - src/registry.rs: defines `FunctionEntry` (using executor's `NativeFunc` type), `inventory::collect!`, the `Registry` singleton (lazily built, sorted by name via `OnceLock`), and `func_table()` to build a Vec<NativeFunc> for the executor
  - src/anim/: split animation native module. `primitives.rs` covers the native Wait/Set/general primitive-animation builders; `std/anim.mcl` exposes that general constructor as `PrimitiveAnim(...)`, while `Lerp(...)` is now just the vanilla rate-driven wrapper with no custom embed/lerp slots. `followers/` now splits the low-level mesh embed/lerp natives by animation family: `mod.rs` keeps the small Grow/Fade/Highlight/Flash embed helpers plus shared helpers and camera lerp, `write.rs` covers Write staging, `trans.rs` covers Trans/TagTrans matching plus shared path-arc interpolation, and `bend.rs` is the thin Bend/TagBend wrapper over that path-arc machinery; the planar `Trans` path now uses explicit LCM loop replication and the trans matcher no longer falls back to the old generic topology conform helper. `std/anim.mcl` now keeps internal mesh-lerp callables private, requires explicit camera candidates for `CameraLerp`, restores `delay{...}` as operator sugar that lowers to an anim block, and expresses `Highlight` / `Flash` as reference-taking compositions over `Set` / `Write` / `Trans` instead of standalone primitive wrappers. `indication.rs` is now just a placeholder module because indication animations are expressed in `std/anim.mcl`; `transfer.rs` covers Transfer/Copy; `composition.rs` covers LaggedMap plus primitive animation operators; `rates.rs` covers bounce/elastic easing; `helpers.rs` owns target resolution, follower/leader mutation, and the remaining shared animation value helpers
  - src/color.rs: color-related native helpers such as `hsv`
  - src/math/: split math native module. `helpers.rs` handles typed argument extraction, including mixed int/float pair reads for scalar natives that can preserve integer results; `scalar.rs` covers elementary math/trig and now keeps integer semantics for cases like integer `mod`, `min`, and `max` instead of promoting eagerly; `combinatorics.rs` covers integer/random helpers, `stats.rs` covers aggregate statistics, `vector.rs` covers dot/cross, and `interpolation.rs` covers general `lerp` plus `keyframe_lerp`
  - src/mesh/: split mesh native module. `helpers.rs` owns mesh-tree traversal, typed mesh/vector readers, primitive mesh builders, callable invocation helpers, transforms, tag-filter support, shared line-chain assembly helpers, topology-slot grouping, indexed line/surface extraction, authored triangle-boundary reconstruction, and libtess2-backed planar loop tessellation; the shared `uprank_mesh` path now centralizes closed-loop promotion and opts into wrapper-side contour normalization before triangulation so redundant duplicate loops do not destabilize tessellation. `constructors.rs` now covers core 2d/3d primitives plus callable-driven graphing/layout helpers, routes planar fills and multi-loop polylines through those shared builders so topology is authored centrally rather than reconstructed ad hoc, interprets sampled domains as `[start, stop, samples]` bundles with descriptive stdlib argument names such as `x_min_max_samples` / `t_min_max_samples`, threads 2d sample indices into `ColorGrid`/`Field` callbacks, accepts scalar-or-pair capsule radii, emits tagged positive/negative regions for `ExplicitFuncDiff`, and enforces hard caps on sampled/grid-generated topology so user code cannot request pathological mesh sizes; `ops.rs` covers styling, remap, extrusion/revolve, and transform operators, with `uprank`/`downrank` now treating closed boundary loops as first-class topology instead of reintroducing fan diagonals by default, with scalar/vector `scale` sharing one native path, with most mesh operators now accepting optional exact-or-predicate tag filters for subset application, and with camera-aware `on_side` / `on_corner` layout helpers replacing the old fixed-bounds side placement; `queries.rs` covers bounds/tag/topology/filter plus sampling/distance/raycast helpers and now includes mesh-tree collapse into a single topology-preserving mesh
  - src/scene.rs: scene/camera helpers backing `std.scene`; provides the default prelude camera plus the default white background scene state, with the public camera constructor surface now centered on `position` + `forward` + `up` + clip range instead of exposing `look_at` directly
  - src/util/: split util native module. `helpers.rs` handles list construction, depth/comparison helpers, key-lambda invocation, and the block-lambda fallthrough runtime error native; `introspect.rs` covers `type_of`, type predicates, dynamic live function/operator attribute helpers (`has_attr` / `get_attr` / `set_attr`), and explicit user-thrown runtime errors; `lists.rs` covers range/sort/zip/slicing/extrema helpers; `maps.rs` covers map iteration helpers; `strings.rs` covers string operations; `convert.rs` covers primitive coercions
- stdlib-macros: proc-macro crate providing the `#[stdlib_func]` attribute; generates a `NativeFunc`-compatible wrapper and submits it to the `inventory` collector used by `stdlib::registry`
- structs: helper structs and utilities.
  - src/assets.rs: contains code for location where assets (say fonts) are located

  - src/rope.rs: a general purpose rope mostly used by the UI for an efficient text editor

- ui: contains all ui related code (everything is gpui)
  - src/components/buttons.rs: contains a button styled like a link
  - src/components/split_pane.rs: gpui element that implements vertical and horizontal split pane
  - src/editor: code related to the text editor. 
    - editor_view.rs: wrapper around `TextEditor`; bootstraps editor content from the internal file, handles periodic internal autosave, writes user saves, and suppresses the pending internal flush when a tab explicitly discards unsaved changes (including pathless scratch tabs that are about to be forgotten)
    - text_editor/mod.rs: contains most of the code related to the actual text editor, which include input handling, trait implementation, action handling, delegating to line reshpaing, and editor-side line transforms such as indent/outdent and command-/ comment toggling across selected rows. Rendering is delegated to other files though.
    - text_editor/popover_element.rs: this is the element that shows editor popovers for diagnostics, autocomplete, and parameter hints; diagnostic popovers also include a clipboard copy action
    - text_editor/text_element: this contains the rendering logic for the actual text editor (including scroll bar and everything on the base layer).
    - wrapped_line.rs: The text editor is implemented to always soft wrap, so this contains the code for rendering a single logical line onto the screen as possibly many lines, with styling. It also allows for e.g. querying the on screen position given a character location.
    - line_shaper.rs: given the lexing data, static analysis data, and diagnostics associated with a single logical line, it applies styling and splits that into the text runs where within each run all characters are styled the same. This effectively creates the data necessary for a wrapped line.
    - line_map.rs: given a bunch of wrapped lines representing the content, it computes the height each line takes up as well as the y position of any logical line number. This allows for various different coordinate system conversions.
  - src/services/{mod.rs,lexing.rs,compilation.rs,execution.rs}: background services that run off the main thread. mod.rs contains ServiceManager and ServiceManagerMessage (including ExecutionSnapshot re-export). `execution.rs` is now a small module root for the executor-thread service types, with the heavy logic split into `src/services/execution/loop_state.rs` (main runtime loop and playback/seek state machine; snapshot-injected runtime errors now cancel further playback/seek/section-advance work so the executor is not asked to cache or advance while already errored; snapshots always publish the current requested target timestamp, the runtime only folds executor time back into that target after successful non-error seek/play advancement, and invalid or out-of-range targets are canonicalized to the concrete reachable scene end before snapshotting so reseeks and scene-end jumps do not make the timeline flash backward or point past the scene), `snapshot.rs` (parameter extraction plus ExecutionSnapshot emission using executor-provided scene snapshots), and `diagnostics.rs` (runtime error formatting). The execution service owns the Executor (Rc-based, !Send), handles UpdateBytecode/SeekTo/TogglePlay/UpdateParameter, and sends ExecutionSnapshot back via ServiceManagerMessage::ExecutionStateUpdated, including current parameter values plus optional stable-state scene data. Background/camera/mesh collection itself now lives in `executor::scene_snapshot`, so UI, CLI, and exporter code can share the same validated capture path; scene data is only attached while execution is `Playing` or `Paused`
  - src/state: code related to global, document, text editor, and similar state
    - diagnostics.rs: errors, warnings, suggestions that are present in the text editor
    - document_state.rs: for a given document, this is just the textual state and the execution state 
    - execution_state.rs: UI-side execution state (current_timestamp, runtime_errors, slide_durations, executor-reported minimum duration hints, current parameter snapshot, slide_count, is_playing, plus viewport camera + background + visible mesh snapshot) updated via ExecutionSnapshot from the executor thread. The UI trusts the executor-thread snapshot timestamp directly; the anti-glitch logic now lives in the execution service, which always publishes the requested target timestamp and only updates that target from executor time after successful non-error execution steps
    - textual_state.rs: this is not only the current text of the document, but also any part of the state of the text editor that might be queryable / editable by other entities in the system (like the current diagnostics, autocomplete). Some other state is owned fully by the text editor instead, in the editor directory. The line is a bit blurry 
    - window_state.rs: global application state, consisting of the active document, open tabs, recently opened projects, and navbar scroll state; owns open/close-tab flows including unsaved-change prompts, discard handling, and forgetting pathless scratch tabs by deleting their internal files
  - src/timeline: a module related to timeline rendering; timeline_view uses exact slide cache durations when available and otherwise falls back to executor-reported minimum duration hints
  - src/viewport: a module related to viewport rendering
    - debug_scene_view.rs: temporary isolated CPU-rendered scene preview; projects mesh snapshots with a parsed camera, now using the simplified forward-vector camera snapshot plus a fixed internal perspective FOV, fills the canvas from the executor-reported background color, paints triangle fills without overlaying triangulation edges, and draws only explicit line/dot primitives into a GPUI canvas until the real shader-backed renderer lands

  - src/navbar_view.rs: navbar element for switching tabs, navigating home, and toggling dark mode
  - src/theme.rs: semantic UI theme definitions, including the GPUI-global persisted `ThemeSettings`, light/dark `ThemeMode`, a white-by-default light palette for documents/viewport/editor, the dark palette, and text editor styling
